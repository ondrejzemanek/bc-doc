BigClown Alpha – Software Setup for Microsoft Windows Host Platform

Prerequisites

  • Microsoft Windows 7, 10 (64bit with CPU virtualization support enabled)

  • Administrator rights

  • Internet connectivity

  • USB cable, Bridge Module and sensors/actuators (Tags, Modules)

Download and install DockerToolbox

  1. Download DockerToolbox-1.12.2.exe, open it and follow installation instructions (you may use Docker’s installation manual).

  2. Allow VirtualBox through Windows Firewall

  3. Start Docker Quickstart Terminal.

  4. Run docker-machine stop and then exit.

  5. Download VirtualBox 5.1.8 Oracle VM VirtualBox Extension Pack and install it

Configure VirtualBox USB and Network

  1. Connect Bridge Module with USB cable.

  2. Start Oracle VM VirtualBox.

  3. Add USB config for FT260 of default virtual machine.

  4. Add Network Port Forwarding in Adapter 1 (NAT) of default virtual machine for HTTP and MQTT.

Configure Docker

  1. Start Docker Quickstart Terminal.

  2. Disconnect Bridge Module from USB.

  3. Connect Bridge Module with USB cable.

  4. Create host directory for Clown.Hub:

    mkdir -v $HOME/hub

Run Clown.Hub software on Microsoft Windows

BigClown uses Docker container technology for seamless software delivery.

Pull Docker image from Docker Hub and run the container in one command:

docker run -d -p 8080:8080 -p 1883:1883 --privileged -v /dev:/dev \
  -v $HOME/hub:/var/hub -h hub-container --restart always --name hub bigclownlabs/hub

You can stop container by:

docker stop hub

And start it again by:

docker start hub

Open the browser and play with the gadgets

Navigate to URL http://hub.local:8080/dashboard or http://localhost:8080/dashboard.

Note
 Be aware that the connection is not encrypted, so it’s not safe to access it over public internet.

MQTT playground

  • Look at measured values (this will subscribe to messages from MQTT broker running inside the container):

    docker exec hub mosquitto_sub -v -t 'nodes/bridge/0/#'

  • Set relay to “true” state (this will publish message to MQTT broker running inside the container):

    docker exec hub \
  mosquitto_pub -t nodes/bridge/0/relay/i2c0-3b/set -m '{"state": true}'

  • Set relay to “false” state:

    docker exec hub \
  mosquitto_pub -t nodes/bridge/0/relay/i2c0-3b/set -m '{"state": false}'

  • If you want to see values from a simulator (running as parallel process inside the container):

    docker exec hub mosquitto_sub -v -t 'nodes/bridge/simulator/#'
